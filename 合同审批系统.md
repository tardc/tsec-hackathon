# CTF 解题记录：合同审批系统

## 赛题信息

- **难度：** Easy
- **分值：** 100
- **描述：** 公司内部合同审批系统上线测试中，用于管理各部门提交的合同文档审批流程。系统区分员工和管理员角色，员工提交合同，管理员负责审批。运维同事提到为了方便集成测试，部分接口做了临时调整。作为安全测试人员，你能发现这些调整带来的风险吗？

## 解题过程

### 第一步：信息收集

访问网站首页，发现这是一个合同审批管理系统，包含登录、合同提交、审批等功能。使用常见凭据 `admin` / `admin123` 成功登录系统。

### 第二步：发现路径遍历漏洞

登录后浏览系统功能，发现 `/download.php` 端点用于下载合同文件。该接口接受一个 `id` 参数来指定要下载的文件。

通过测试发现，`download.php` 存在 **路径遍历漏洞 (Path Traversal, CWE-22)**：
- 代码将 `id` 参数直接拼接到合同目录路径 `/var/www/html/contracts/` 后面
- 没有对 `../` 等路径遍历字符进行过滤或转义

### 第三步：利用漏洞读取敏感文件

利用路径遍历，可以读取服务器上的任意文件：

1. **读取配置文件**，获取数据库凭据和系统信息：
   ```
   /download.php?id=../../../../var/www/html/includes/config.php
   ```

2. **读取 Flag 文件**：
   ```
   /download.php?id=../../../../challenge/flag.txt
   ```

通过 `../` 不断向上跳转目录，最终从 `/var/www/html/contracts/` 跳到根目录 `/`，然后进入 `/challenge/flag.txt` 读取 Flag。

### 关键代码分析

`download.php` 中的核心漏洞代码大致如下：
```php
$file = '/var/www/html/contracts/' . $_GET['id'];
readfile($file);  // 直接读取，没有任何过滤
```

正确的做法应该是对 `id` 参数进行白名单校验或使用 `basename()` 函数来防止路径遍历。

## 涉及的知识点

1. **路径遍历漏洞 (Path Traversal / Directory Traversal)：** 通过 `../` 序列突破预期的目录限制，访问服务器上的任意文件
2. **不安全的文件操作：** 直接将用户输入拼接到文件路径中，没有进行输入验证和过滤
3. **PHP 安全：** `readfile()` 等函数如果接受用户可控的路径参数，必须进行严格的白名单校验

## 防御建议

- 对文件下载接口使用白名单校验，只允许访问特定目录下的文件
- 使用 `basename()` 函数去除路径中的目录部分
- 对用户输入进行过滤，移除 `../`、`..\\` 等路径遍历字符
- 使用 chroot 或限制 PHP 进程的文件系统访问权限
