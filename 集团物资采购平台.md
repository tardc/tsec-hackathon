# CTF 解题记录：集团物资采购平台

## 赛题信息

- **难度：** Easy
- **分值：** 100
- **描述：** 集团内部物资申购系统V2.0。S级特权凭证(Flag)标价高达9999元，而初始经费只有100元，正常情况下根本买不起。需要找到漏洞绕过价格限制。

## 解题过程

### 第一步：信息收集

访问网站后自动跳转到 `/login` 登录页面。观察登录表单发现密码字段是 `disabled` 状态，说明**只需要用户名即可登录**，无需密码。

使用任意用户名（如 `admin`）POST 到 `/login` 即可成功登录获取 session。

### 第二步：分析商店功能

登录后进入商店页面，可以看到 4 件商品：

| 商品 | 价格 | product_id |
|------|------|------------|
| 普通办公椅 | 10 元 | 1 |
| 机械键盘 | 50 元 | 2 |
| 降噪耳机 | 120 元 | 3 |
| **S级特权凭证 (Flag)** | **9999 元** | 8848 |

初始余额为 100 元，远不足以购买 9999 元的 Flag 商品。

### 第三步：发现并利用漏洞

购买接口为 `POST /buy`，参数包括：
- `product_id` — 商品ID
- `buy_qty` — 购买数量

**漏洞：** 服务器没有对 `buy_qty` 参数进行正整数校验，允许传入负数。

**利用步骤：**

1. **用负数数量购买 Flag 商品**，将 `buy_qty` 设为 `-1`：
   ```
   POST /buy
   product_id=8848&buy_qty=-1
   ```
   计算逻辑：`-1 × 9999 = -9999`，即从余额中**扣除 -9999 元**，相当于**增加 9999 元**。
   此时余额变为：`100 + 9999 = 10099` 元。

2. **用正常数量再次购买**：
   ```
   POST /buy
   product_id=8848&buy_qty=1
   ```
   扣除 9999 元后，成功获得 Flag 商品。

### 漏洞原理

这是一个典型的 **整数符号验证缺失** 漏洞。购买逻辑大致如下：

```python
total = price * buy_qty
balance = balance - total
```

当 `buy_qty = -1` 时：
```
total = 9999 * (-1) = -9999
balance = 100 - (-9999) = 10099
```

因为服务端没有验证 `buy_qty > 0`，负数导致扣款变加款。

## 涉及的知识点

1. **输入验证缺失：** 服务端没有对用户输入的数值进行类型和范围校验
2. **业务逻辑漏洞：** 购买数量允许为负数，导致金额计算出现反转
3. **整数符号问题：** 在涉及金额计算的逻辑中，必须确保数值为正整数

## 防御建议

- 对所有用户输入进行严格的类型和范围校验（购买数量必须为正整数）
- 在服务端业务逻辑层添加金额上限校验
- 对关键操作（如购买、转账）添加二次确认和日志记录
- 使用 decimal 类型而非浮点数进行金额计算，避免精度问题
