# CTF 解题记录：文件签名审计 (OceanLink)

## 赛题信息

- **难度：** Easy
- **分值：** 100
- **描述：** OceanLink 是公司内部的新一代文件管理系统。系统对所有敏感文件请求都施加了特殊的"安全签名"，以防止攻击者篡改文件名获取未授权数据。你需要通过逆向 JS 核心逻辑，还原出签名算法并读取 flag.txt。

## 解题过程

### 第一步：信息收集

用浏览器或 curl 访问网站首页，发现这是一个名为 OceanLink 的文件管理系统。页面使用了 MD5 签名来保护文件访问权限。查看页面源码，发现加载了两个关键的 JS 文件：
- `/static/js/index.js` — 主逻辑文件
- `md5.min.js` — MD5 哈希库

### 第二步：下载并分析 JS 文件

下载 `index.js` 进行分析。虽然代码经过了混淆处理，但逻辑仍然可以还原：

1. **找到盐值（Salt）拼接逻辑：**
   - 混淆代码中有一个映射表，包含了以下字符串的十六进制编码：
     - `Oce@n`
     - `L1nk`
     - `_`
     - `S3cr3t`
     - `2024`
   - `assembleSecurityShield()` 函数将它们按顺序拼接，得到完整的盐值：
     ```
     Oce@nL1nk_S3cr3t_2024
     ```

2. **还原签名算法：**
   - `generateFileSignature(filename)` 函数的逻辑非常简单：
     ```
     签名 = MD5(文件名 + 盐值)
     ```
   - 也就是说，对于文件 `flag.txt`，签名为：
     ```
     MD5("flag.txt" + "Oce@nL1nk_S3cr3t_2024")
     ```

### 第三步：生成签名

使用任意 MD5 计算工具，计算签名：

```
md5("flag.txtOce@nL1nk_S3cr3t_2024") = 64cf30decb95dea8792c5b7db45d13ea
```

### 第四步：获取认证

通过 `POST /login` 接口注册/登录，获取 session cookie。系统会自动为新用户创建账号。

### 第五步：读取 Flag

带上 session cookie，使用正确的签名参数请求文件接口：

```
GET /api/file?name=flag.txt&sig=64cf30decb95dea8792c5b7db45d13ea
Cookie: session=<your_session>
```

成功返回 Flag 内容。

## Flag

```
flag{16ccc11c36c5fffda860bf1cca8c6fda}
```

## 涉及的知识点

1. **JavaScript 逆向工程：** 如何分析和还原混淆后的 JS 代码
2. **客户端签名机制的安全缺陷：** 签名盐值硬编码在前端代码中，攻击者可以直接还原
3. **MD5 哈希：** 了解 MD5 签名的计算方式

## 防御建议

- 不要在前端代码中暴露签名密钥/盐值
- 签名验证应该在后端完成，使用服务端密钥
- 对敏感文件的访问应该基于服务端的权限控制，而非客户端签名
